Preface

It turned out that many software components initially conceived for DCS found applications in non-DCS groups, e.g. in TDAQ or other forms of configuration software.

At the time of writing (Feb 2020) this applies mostly to:

* open62541-compat
* UaObjects client for SCA OPC-UA
* LogIt

Quasar normally takes good care of integrating these software components, so as long as you develop quasar server(s), you are good and you do not have to think too much.

However, what happens when you abandon Quasar? Then you have to think how to deploy these dependencies.

This project is precisely made for that.

Steps


OK, nothing fancy really:

  • setup your compiler (source or set paths of whatever). After this step, invoking "c++" should be routed to the compiler of choice. For example, in my case:
    $ c++ --version
    c++ (GCC) 8.3.0
  • decide where you'd like to deploy the software built by this script.
    I (Piotr) normally deploy it to /opt but this might vary.
    Note that deployment will be done by equivalent of "make install" so make sure that the target location is writable, etc.
    The path you decided on will be used in the step below.
  • just run the scripts, e.g.:
    ./prepare.sh --install_prefix /opt/tdaqQuasarEcosystem/gcc8.3.0
